# homelab-go

Этот репозиторий является корневым-проектом, объединяющим несколько Go‑сервисов через Git submodules.  
Каждый сервис хранится в собственном репозитории и подключается как подмодуль, что позволяет упрощать локальную разработку.

## Структура

- docker-compose.yml – запуск всей инфраструктуры
- Makefile – команды для удобного упревления сервисами
- simple-service - простой тестовый сервис
- ~~auth-service/ – сервис аутентификации~~
- ~~api-gateway/ – API‑шлюз~~
- ~~scripts/ – вспомогательные скрипты~~

## Клонирование проекта

Репозиторий содержит подмодули, поэтому при клонировании обязательно используйте флаг --recurse-submodules:
```
git clone --recurse-submodules git@github.com:Pypurka1337/homelab-go.git
```

Если вы уже склонировали репозиторий без подмодулей, выполните:
```
git submodule update --init --recursive
```

## Обновление подмодулей

Подмодули настроены на отслеживание определённых веток (например, main).
Чтобы обновить все подмодули до последних коммитов из этих веток, выполните в корне проекта:

```
git submodule update --remote --recursive
```

После обновления не забудьте зафиксировать новые версии подмодулей в супер‑проекте:

```
git add .
git commit -m "Обновление подмодулей до последних версий"
```

## Работа с общим кодом

Если сервисы используют общие библиотеки, они вынесены в отдельный подмодуль common/.
В go.mod каждого сервиса добавлена директива replace, указывающая на локальный путь:

```
replace github.com/Pypurka1337/common => ../common
```
Это позволяет разрабатывать общий код и сервисы одновременно без публикации изменений в удалённый репозиторий.

## Полезные команды
* `git status` – покажет, какие подмодули имеют незакоммиченные изменения или отличаются от зафиксированной версии.
* `git diff --submodule` – покажет изменения внутри подмодулей.
* `git submodule foreach 'git status'` – выполнит команду в каждом подмодуле.
